# Работа с трекером (Vikunja)

Пайплайн: `Backlog → Queue → Design → Build → Review → [человек] → Done`,
плюс отдельная колонка `Call to Human`. Жёсткие правила зашиты в MCP-тулзы —
они откажут, если что-то не так. Эти правила — про то, КАК работать.

## Дисциплина очереди

- **Одна задача за раз.** Новую не клеймить, пока текущая не в Review,
  Call to Human или не возвращена через return_task.
- Начало работы с задачами — всегда `next_task`: сначала он вернёт твою
  активную задачу (в т.ч. вернувшуюся после ответа человека), потом очередь.
- Backlog не трогаем — это зона триажа человека.

## Следы работы (комменты — это журнал)

- При клейме тулза сама отметится; следом опиши план коротким `comment`.
- Фиксируй находки и решения по ходу: «выбрал X вместо Y, потому что Z»,
  «наступил на гочу W» — это читают и люди, и следующие агенты.
- `advance(to='build')` требует spec — 2-5 предложений о подходе, не эссе.
- `advance(to='review')` требует worklog (что сделано) + evidence —
  ссылку на коммит/PR или вывод верификации. Прогони проверку ДО перевода.

## Декомпозиция

- Задача больше ~половины дня работы или из нескольких несвязных тем —
  `decompose` на подзадачи (каждая — самостоятельно проверяемый результат).
  Подзадачи встанут в Queue, родитель уедет эпиком в Backlog.

## Застрял? Два разных выхода

- **`call_human`** — нужно решение или вводные от человека (выбор из вариантов,
  доступ к секрету, продуктовое решение). Задача остаётся твоей. В вопросе:
  что нужно, какие варианты рассмотрел, что рекомендуешь.
  После ответа человек вернёт задачу в Design/Build — она снова придёт
  через next_task как «твоя активная».
- **`return_task`** — внешняя блокировка (чужой сервис лежит, нет зависимости,
  задача потеряла смысл). Клейм снимается, задача уходит в Backlog на ре-триаж.

## После Review

- Задачу в Review не трогай. Правки по ревью = человек вернул её в Build
  с комментом — увидишь через next_task, прочитай комменты через get_task.
- В Done переводит только человек. Никогда не пытайся обойти это через API.
